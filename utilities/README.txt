# IAC80 Photometry and Telemetry Analysis Scripts
# =========================================================
# This repository contains a collection of Python scripts designed to analyze, 
# monitor, and evaluate image quality (specifically PSF/FWHM) and telemetry 
# data from FITS files.
#
# Below is an index and description of each script, including their operational 
# relationships and version histories.

# ---------------------------------------------------------
# 1. LIVE MONITORING & DASHBOARD
# ---------------------------------------------------------

alpha_html.py
    The flagship live telemetry script. It acts as a watchdog monitoring an 
    incoming data directory for new FITS files. When a file arrives, it performs 
    photometry using a custom Moffat2D model, computes guiding errors, and 
    extracts telemetry from the headers. It hosts a live interactive web 
    dashboard (using Flask and Plotly) for real-time visualization at the telescope.

auto_photometry.py
    [OLD VERSION / PRECURSOR to alpha_html.py] 
    An earlier, simpler version of the real-time watchdog script. Instead of 
    hosting a live web dashboard, it generates and saves static matplotlib 
    images (histograms of FWHM, Phi, and Ellipticity) to a results folder for 
    each incoming FITS file.

simulate_telescope.py
    A utility testing script. It copies existing FITS files one-by-one from a 
    source directory into an "incoming" directory with a simulated delay. This 
    is used to test the watchdog functionality of `alpha_html.py` or 
    `auto_photometry.py` without needing a real active telescope session.


# ---------------------------------------------------------
# 2. BATCH PROCESSING & ARCHIVE ANALYSIS
# ---------------------------------------------------------

parallel_csv.py
    A high-performance batch processing script. It uses multiprocessing (`joblib`) 
    to iterate through historical FITS data archives. It applies the custom Moffat 
    PSF model to calculate photometry, FWHM, and other metadata, saving the 
    results into daily CSV files.

quality_csv.py
    Reads the nightly CSV files (generated by parallel_csv.py) for a specific 
    observation directory. It calculates a weighted "Image Quality Score" based 
    on FWHM, ellipticity, and field consistency, and generates time-series panel 
    plots handling large date gaps dynamically.

all_years_csv.py
    An advanced extension of `quality_csv.py`. Instead of analyzing a single 
    folder, it recursively searches for all generated CSVs across multiple years. 
    It aligns the dates and generates smoothed 14-day trend lines comparing 
    Image Quality Scores and Overall FWHM year-over-year.

csv_to_plot.py
    A simple standalone visualization script. It reads a specifically targeted 
    CSV file (e.g., a region of interest) and generates a stacked 5-panel plot 
    comparing FWHM against environmental/instrumental telemetry (Temperature, 
    Humidity, Airmass, and Telescope Focus).


# ---------------------------------------------------------
# 3. SINGLE-IMAGE EXPLORATION & MODEL COMPARISON
# ---------------------------------------------------------

one_input_Gaussian.py
    [EXPLORATORY SCRIPT] 
    Analyzes a single specific FITS file. It uses a standard Gaussian PSF model 
    to extract FWHM/ellipticity and saves 5 analytical plots (2D map, model 
    comparison, statistics, trends, and residuals).

one_input_Moffat.py
    [EXPLORATORY SCRIPT] 
    Identical in structure to the Gaussian script above, but implements the 
    custom 2D Elliptical Moffat model instead. 

one_input_Residuals.py
    [FINAL STANDALONE COMPARISON SCRIPT]
    The culmination of the previous two scripts. It explicitly targets two specific 
    sources in a single FITS image (one with coma aberration, one round) and 
    applies BOTH the Gaussian and Moffat models. It generates side-by-side 
    residual maps and 1D profile cuts (X and Y axes) to visually demonstrate 
    why the Moffat model is a superior fit.
